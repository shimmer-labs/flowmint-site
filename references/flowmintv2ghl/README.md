# Rules directory

Domain rules go here, added as pain points emerge, NOT upfront. The discipline: when you have explained the same thing to the agent three times, make it a rule.

Examples of what would land here over time:
- A specific GHL API quirk the agent keeps getting wrong.
- A code-style pattern it keeps drifting from.
- A repeated mistake in template structure.

Keep CLAUDE.md for always-on, repo-wide context. Keep rules for filtered, task-specific domain knowledge. Do not pre-populate; let real friction drive what gets added.
